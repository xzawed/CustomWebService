지정된 파일 또는 모듈에 대한 포괄적인 테스트를 생성하세요.

대상: $ARGUMENTS

## 테스트 생성 절차

### 1. 대상 파일 분석
- 대상 파일을 읽고 모든 public 함수/메서드 파악
- 의존성(import) 확인 — 모킹 대상 식별
- 에러 경로 및 엣지 케이스 파악

### 2. 테스트 파일 생성
- **위치**: 소스 파일 옆 co-located (`{파일명}.test.ts`) 또는 `src/__tests__/`
- **프레임워크**: Vitest (`describe`, `it`, `expect`)
- **환경**: `happy-dom` (컴포넌트) 또는 기본 Node (서비스/유틸)

### 3. 테스트 구성
```
describe('{클래스/함수명}', () => {
  // 정상 동작 테스트
  describe('성공 케이스', () => { ... })

  // 에러/예외 테스트
  describe('실패 케이스', () => { ... })

  // 엣지 케이스
  describe('엣지 케이스', () => { ... })
})
```

### 4. 모킹 규칙
- Supabase 클라이언트: `vi.mock('@/lib/supabase/server')` 또는 MSW
- 외부 API: MSW 핸들러 사용
- 다른 서비스/리포지토리: `vi.mock()` 사용
- **절대 DB를 직접 호출하지 않음** (단위 테스트)

### 5. 참고 파일
- `src/__tests__/` 내 기존 테스트 파일 패턴 확인
- `src/test/` 내 테스트 헬퍼/설정 확인
- `vitest.config.ts` 설정 확인

### 6. 검증
- 생성 후 `pnpm test` 실행하여 테스트 통과 확인
