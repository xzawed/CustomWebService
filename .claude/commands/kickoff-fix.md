버그 수정을 위한 체계적 디버깅 워크플로우를 실행하세요.

## 버그 설명
$ARGUMENTS

## 워크플로우

### Step 1: 조사 (Explore 에이전트 병렬)
Agent 도구로 Explore 타입 에이전트를 병렬 실행:
- **에이전트 A**: 버그 관련 코드 경로 추적 (에러 메시지, 스택 트레이스 기반)
- **에이전트 B**: 관련 테스트 파일 확인 — 기존 테스트가 이 케이스를 커버하는지
- **에이전트 C**: 최근 git 변경사항 확인 — 버그를 유발했을 수 있는 커밋

### Step 2: 근본 원인 분석 (직접 수행)
에이전트 결과를 종합하여:
1. 버그의 근본 원인 식별
2. 영향 범위 파악 (같은 패턴의 다른 코드도 영향받는지)
3. 수정 방안 도출 (최소 변경 원칙)
4. 사용자에게 분석 결과 보고

### Step 3: 수정 구현
- 최소한의 변경으로 버그 수정
- 관련 파일만 수정 (불필요한 리팩토링 금지)

### Step 4: 회귀 테스트 추가
- 이 버그를 재현하는 테스트 케이스 추가
- 수정 후 테스트 통과 확인

### Step 5: 코드 검증
```bash
pnpm lint
pnpm type-check
pnpm test
```

### Step 6: 영향 범위 추적 (필수 — 생략 금지)
이 서비스는 다수 사용자가 이용 중입니다.
- 수정한 코드가 호출되는 **모든 경로**를 나열하고 각 경로에서 정상 동작하는지 추적
- 같은 함수를 사용하는 다른 경로(미리보기, 게시, 서브도메인 등)에 부작용이 없는지 확인
- 미들웨어 → route handler → 브라우저까지 전체 흐름에서 헤더 이중 적용 등 확인
- ❌ 항목이 있으면 커밋하지 마세요

수정 내용과 영향 범위를 ✅/❌ 형태로 요약 보고하세요.
