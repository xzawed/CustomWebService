코드 리팩토링을 위한 안전한 워크플로우를 실행하세요.

## 리팩토링 대상/목표
$ARGUMENTS

## 워크플로우

### Step 1: 현황 분석 (Explore 에이전트 병렬)
Agent 도구로 Explore 타입 에이전트를 병렬 실행:
- **에이전트 A**: 리팩토링 대상 코드 전체 읽기 + 의존성 그래프 파악
- **에이전트 B**: 대상 코드를 참조하는 모든 곳 탐색 (import, 호출 지점)
- **에이전트 C**: 관련 테스트 커버리지 확인

### Step 2: 리팩토링 계획 (Plan 에이전트)
Agent 도구로 Plan 타입 에이전트 실행:
- 변경 범위 및 영향도 분석
- 단계별 리팩토링 계획 (각 단계가 독립적으로 빌드 가능하도록)
- 위험 요소 식별
- 사용자에게 계획 보고 후 승인 요청

### Step 3: 기존 테스트 실행 (기준선)
```bash
pnpm test
```
현재 테스트가 모두 통과하는지 기준선 확인

### Step 4: 리팩토링 실행
계획에 따라 단계별 실행:
- 각 단계 완료 후 `pnpm type-check` 실행
- 타입 에러 발생 시 즉시 수정
- import 경로, export 변경 시 모든 참조 업데이트

### Step 5: 테스트 업데이트
- 변경된 인터페이스에 맞게 기존 테스트 수정
- 새로운 구조에 대한 테스트 추가 (필요시)

### Step 6: 최종 검증
```bash
pnpm lint
pnpm type-check
pnpm test
pnpm build
```

리팩토링 전후 비교 및 개선 사항을 요약 보고하세요.
