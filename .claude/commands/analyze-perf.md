## 분석 항목

### 1. 데이터베이스 쿼리
- N+1 쿼리 패턴 탐지
- 불필요한 SELECT * 사용
- 인덱스 미활용 쿼리
- 트랜잭션 범위 적절성
- Supabase RPC 함수 활용 가능성

### 2. React 렌더링 (컴포넌트 대상)
- 불필요한 리렌더링 발생 지점
- `useMemo`, `useCallback` 필요 여부
- 컴포넌트 분할로 렌더링 범위 축소 가능 여부
- Zustand 스토어 구독 범위 (selector 최적화)

### 3. API 호출
- 중복 요청 발생 가능성
- 캐싱 전략 부재
- 응답 데이터 크기 최적화 (필요한 필드만 반환)
- SSE/스트리밍 활용 적절성

### 4. 번들 사이즈
- 불필요한 import (tree-shaking 방해)
- 동적 import(`next/dynamic`) 활용 가능성
- 클라이언트/서버 컴포넌트 분리 적절성

### 5. 개선 제안
- 각 문제에 대해 **현재 코드** vs **개선 코드** 비교
- 예상 개선 효과 설명
- 우선순위(높음/중간/낮음) 분류
