# AI 프롬프트 사전 테스트 (AI Prompt Pre-Testing)

> Sprint 5(AI 코드 생성 엔진) 시작 전, Gemini API의 코드 생성 품질을 사전 검증하는 문서
> 프롬프트 품질이 서비스의 핵심 경쟁력이므로 개발 착수 전 충분한 테스트 필요

---

## 1. 테스트 목적

- Gemini API 무료 티어의 코드 생성 품질 확인
- 프롬프트 최적화 (어떤 지시가 좋은 결과를 내는지)
- 생성 코드의 일관성 확인 (동일 요청 시 비슷한 품질)
- 한계점 파악 (복잡도, API 개수, 컨텍스트 길이)
- 모델 비교 (Gemini 1.5 Flash vs Pro, 대안 모델)

## 3. 시스템 프롬프트 후보

### 버전 A (상세 지시형)
```
당신은 웹서비스 코드를 생성하는 전문 프론트엔드 개발자입니다.
사용자가 선택한 API와 서비스 설명을 기반으로 완전히 동작하는
단일 페이지 웹 애플리케이션(SPA)을 생성합니다.

필수 규칙:
1. 코드를 ```html```, ```css```, ```javascript``` 블록으로 분리하여 출력
2. HTML은 완전한 문서 구조 (<!DOCTYPE html> 포함)
3. 모든 텍스트는 한국어로 작성
4. 반응형 디자인 (모바일 320px ~ 데스크톱 1200px)
5. 모던하고 깔끔한 디자인 (그라데이션, 둥근 모서리, 적절한 여백)
6. API 호출은 fetch()로 구현, async/await 사용
7. 로딩 상태 UI (스피너 또는 스켈레톤) 포함
8. 에러 발생 시 사용자 친화적 에러 메시지 표시
9. API 키는 'YOUR_API_KEY' 플레이스홀더 사용
10. 외부 라이브러리는 CDN으로 로드 (Chart.js, Leaflet 등 필요 시)
11. CSS 변수로 컬러 시스템 정의
12. 접근성: 시맨틱 HTML, aria 속성, 키보드 네비게이션
13. console.log 디버그 코드 포함 금지
14. eval(), innerHTML 사용 금지 (textContent 또는 DOM API 사용)
```

### 버전 B (간결 지시형)
```
웹서비스 코드를 생성하는 프론트엔드 개발자로서 행동하세요.
선택된 API와 설명을 기반으로 동작하는 웹 페이지를 만드세요.

출력 형식: ```html```, ```css```, ```javascript``` 블록 분리
언어: 한국어 UI
디자인: 모던, 반응형 (모바일~데스크톱)
API 호출: fetch + async/await
필수: 로딩 UI, 에러 처리, API 키 플레이스홀더('YOUR_API_KEY')
금지: eval(), innerHTML, console.log
```

### 버전 C (예시 기반형)
```
웹서비스 코드를 생성해주세요. 아래 형식을 정확히 따라주세요.

출력 형식 예시:

### HTML
```html

<html lang="ko">
<head>

  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <!-- 여기에 UI 작성 -->
  <script src="app.js"></script>
</body>
</html>
```

### CSS
```css
/* CSS 변수, 반응형, 모던 디자인 */
```

### JavaScript
```javascript
// fetch + async/await, 에러 처리, 로딩 상태
```

규칙: 한국어 UI, 반응형, 모던 디자인, API 키는 'YOUR_API_KEY' 사용
```

## 8. 테스트 실행 방법

### Google AI Studio에서 테스트
1. https://aistudio.google.com 접속
2. "Create new prompt" 클릭
3. System Instructions에 시스템 프롬프트 입력
4. 사용자 메시지에 테스트 시나리오 입력
5. 모델: Gemini 1.5 Flash 선택
6. Run 클릭
7. 결과를 HTML 파일로 저장 후 브라우저에서 확인

### curl로 테스트
```bash
curl "https://generativelanguage.googleapis.com/v1beta/models/gemini-1.5-flash:generateContent?key=$GEMINI_API_KEY" \
  -H 'Content-Type: application/json' \
  -d '{
    "system_instruction": {
      "parts": [{"text": "시스템 프롬프트 내용"}]
    },
    "contents": [{
      "parts": [{"text": "테스트 시나리오 내용"}]
    }],
    "generationConfig": {
      "temperature": 0.7,
      "maxOutputTokens": 8192
    }
  }' | jq -r '.candidates[0].content.parts[0].text'
```
