# UI/UX 설계 (UI/UX Design Specification)

## 1. 디자인 원칙

- **심플**: 비개발자도 이해할 수 있는 직관적 인터페이스
- **가이드**: 단계별 안내로 사용자가 길을 잃지 않도록
- **피드백**: 모든 액션에 즉각적인 시각적 피드백
- **접근성**: WCAG 2.1 AA 기준 준수

---

**인터랙션 상세**:
- 왼쪽 카탈로그에서 체크박스 클릭 또는 드래그 → 오른쪽 선택 영역으로 이동
- 선택된 API 카드: 삭제(X) 버튼으로 제거
- API 카드 클릭 시 상세 정보 모달 (제공 데이터, 호출 제한, 예시)
- 카테고리 필터: 탭 또는 드롭다운

## 3. 컴포넌트 명세

### 3.1 API 카드 컴포넌트
```
┌────────────────────────────────┐
│ ☐  🌤 OpenWeatherMap          │
│                                │
│ 현재 날씨, 5일 예보, 대기질      │
│                                │
│ 🏷 날씨  ⏱ 1,000회/일  🔑 Key │
│                                │
│ [자세히 보기]                   │
└────────────────────────────────┘
```

Props:
- `id`: API ID
- `name`: API 이름
- `description`: 설명
- `category`: 카테고리
- `rateLimit`: 호출 제한
- `authType`: 인증 방식
- `iconUrl`: 아이콘
- `isSelected`: 선택 여부
- `onSelect`: 선택 핸들러
- `draggable`: 드래그 가능 여부

### 3.2 컨텍스트 입력 컴포넌트
- 텍스트 에어리어 (자동 높이 조절)
- 글자 수 카운터 (50~2000자)
- 가이드 질문 토글
- 템플릿 선택 버튼 그룹

### 3.3 생성 진행 컴포넌트
- 단계별 프로그레스 바
- 각 단계 상태 아이콘 (대기/진행중/완료/실패)
- 예상 소요 시간 표시
- 생성 로그 실시간 표시 (선택)

## 4. 반응형 브레이크포인트

| 화면 | 너비 | 레이아웃 |
|------|------|---------|
| 모바일 | < 640px | 단일 컬럼, 체크박스만 사용 |
| 태블릿 | 640px ~ 1024px | 2컬럼, DnD + 체크박스 |
| 데스크톱 | > 1024px | 2컬럼 사이드바, DnD 기본 |
