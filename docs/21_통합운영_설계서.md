# 통합 운영 설계서 — SuperGrok + n8n Cloud + Claude CLI

> 3자 통합 자동화 운영 체계의 전체 설계, 워크플로우 구성, 연결 사양.

## 1. 아키텍처 개요

```
┌─────────────────────┐
│   xzawed.xyz        │
│   (Railway)         │
│                     │
│ WebhookForwarder ───┼──→ n8n Cloud (웹훅 수신)
│ Admin API ←─────────┼──← n8n Cloud (데이터 조회)
│ Health API ←────────┼──← n8n Cloud (상태 확인)
└─────────────────────┘
         ▲
         │ GitHub Actions
         │
┌────────┴────────────────────────────────────────┐
│                  n8n Cloud                       │
│              (오케스트레이션)                      │
│                                                  │
│  ┌─ 웹훅 수신 ─→ 이벤트 라우팅 ─→ 판단 분기     │
│  │                                               │
│  ├─→ SuperGrok (xAI API) ── 분석/리뷰/분류       │
│  ├─→ Claude CLI (GitHub Actions) ── 코드 수정    │
│  ├─→ Discord/Slack ── 알림                       │
│  └─→ Supabase ── 직접 데이터 조회                │
└──────────────────────────────────────────────────┘
```

## 2. 환경변수 설정

### Railway (앱 서버)

| 변수 | 값 | 설명 |
|------|-----|------|
| `N8N_WEBHOOK_URL` | `https://<instance>.app.n8n.cloud/webhook/app-events` | n8n Cloud 웹훅 URL |
| `N8N_WEBHOOK_SECRET` | 64자 랜덤 문자열 | HMAC-SHA256 서명 키 |
| `ADMIN_API_KEY` | 64자 랜덤 문자열 | 관리자 API 인증 키 |
| `ENABLE_RENDERING_QC` | `true` | 렌더링 QC 활성화 |

### n8n Cloud (크리덴셜로 저장)

| 크리덴셜 이름 | 타입 | 값 |
|-------------|------|-----|
| `App Admin API` | Header Auth | `Authorization: Bearer <ADMIN_API_KEY>` |
| `xAI API` | Header Auth | `Authorization: Bearer <XAI_API_KEY>` |
| `GitHub PAT` | Header Auth | `Authorization: Bearer <GITHUB_TOKEN>` |

|--------|------|
| `ANTHROPIC_API_KEY` | Claude CLI 실행 (이미 존재할 수 있음) |

## 3. 웹훅 수신 사양

### 앱 → n8n 웹훅 페이로드

```json
{
  "type": "이벤트_타입",
  "payload": { ... },
  "timestamp": "2026-04-05T09:00:00.000Z"
}
```

**헤더:**
```
Content-Type: application/json
X-Webhook-Signature: sha256=<HMAC_SHA256_HEX>
```

**서명 검증 (n8n Code 노드):**
```javascript
const crypto = require('crypto');
const secret = $credentials.webhookSecret;
const body = JSON.stringify($input.all()[0].json);
const expected = crypto.createHmac('sha256', secret).update(body).digest('hex');
const received = $input.all()[0].headers['x-webhook-signature']?.replace('sha256=', '');
if (expected !== received) throw new Error('Invalid signature');
```

### 이벤트 타입 전체 목록

| 이벤트 | 발생 시점 | 핵심 페이로드 |
|--------|----------|-------------|
| `CODE_GENERATED` | 코드 생성 성공 | projectId, version, provider, durationMs |
| `CODE_GENERATION_FAILED` | 코드 생성 실패 | projectId, error, provider |
| `QC_REPORT_COMPLETED` | QC 검증 완료 | projectId, overallScore, passed, checks[], isDeep |
| `QC_REPORT_FAILED` | QC 검증 오류 | projectId, stage(fast/deep), error |
| `DEPLOYMENT_STARTED` | 배포 시작 | projectId, platform |
| `DEPLOYMENT_COMPLETED` | 배포 완료 | projectId, url, platform |
| `DEPLOYMENT_FAILED` | 배포 실패 | projectId, error |
| `PROJECT_PUBLISHED` | 사이트 게시 | projectId, userId, slug |
| `PROJECT_UNPUBLISHED` | 게시 해제 | projectId, userId |
| `PROJECT_CREATED` | 프로젝트 생성 | projectId, userId, apiCount |
| `PROJECT_DELETED` | 프로젝트 삭제 | projectId |
| `USER_SIGNED_UP` | 신규 가입 | userId |
| `API_QUOTA_WARNING` | API 쿼터 경고 | service, usage, limit |

  Authorization: Bearer <ADMIN_API_KEY>

      { "check": "touchTargets", "failCount": 34, "rate": 0.24 },

## 5. n8n Cloud 워크플로우 설계

### Workflow A: QC 실패 → 자동 수정

```
트리거: 웹훅 (app-events)
조건: type === 'QC_REPORT_COMPLETED' && passed === false && isDeep === true

[1. Webhook Trigger]
  │ URL: /webhook/app-events
  │ Method: POST
  ▼
[2. IF Node: QC 실패 여부]
  │ 조건: $json.payload.passed === false
  │ AND: $json.payload.isDeep === true (Deep QC만 — Fast는 파이프라인 내에서 자동 처리됨)
  ▼
[3. HTTP Request: QC 통계 조회]
  │ GET https://xzawed.xyz/api/v1/admin/qc-stats?days=7
  │ Auth: App Admin API 크리덴셜
  │ → 최근 트렌드와 함께 분석 요청
  ▼
[4. HTTP Request: SuperGrok 분석]
  │ POST https://api.x.ai/v1/chat/completions
  │ Auth: xAI API 크리덴셜
  │ model: "grok-3"
  │ messages: [{ role: "user", content: QC_ANALYSIS_PROMPT }]
  │
  │ 프롬프트에 포함:
  │   - QC 실패 상세 (checks 배열)
  │   - 최근 7일 트렌드 (qc-stats 결과)
  │   - "자동 수정 가능 여부와 구체적 수정 지시를 JSON으로 반환"
  ▼
[5. Code Node: SuperGrok 응답 파싱]
  │ JSON.parse(response.choices[0].message.content)
  │ → { autoFixable, fixApproach, severity, affectedArea }
  ▼
[6. IF Node: 자동 수정 가능?]
  │
  ├── autoFixable === true:
  │   ▼
  │   [7. HTTP Request: GitHub repository_dispatch]
  │   │ POST https://api.github.com/repos/xzawed/CustomWebService/dispatches
  │   │ Auth: GitHub PAT 크리덴셜
  │   │ Body:
  │   │ {
  │   │   "event_type": "claude-fix",
  │   │   "client_payload": {
  │   │     "task_description": "{{ SuperGrok fixApproach }}",
  │   │     "commit_message": "fix: QC 자동 수정 — {{ affectedArea }}",
  │   │     "pr_title": "[Auto] QC Fix: {{ affectedArea }}",
  │   │     "pr_body": "## QC 분석 결과\n{{ SuperGrok 전체 분석 }}\n\n## 수정 대상\n- 프로젝트: {{ projectId }}\n- QC 점수: {{ overallScore }}/100"
  │   │   }
  │   │ }
  │   ▼
  │   [8. Discord: 알림]
  │   │ "🔧 QC 자동 수정 시작 — 프로젝트 {{ projectId }}, 점수 {{ overallScore }}"
  │
  └── autoFixable === false:
      ▼
      [9. HTTP Request: GitHub Issue 생성]
      │ POST https://api.github.com/repos/xzawed/CustomWebService/issues
      │ title: "[QC] {{ severity }}: {{ summary }}"
      │ body: SuperGrok 분석 + 수동 조치 필요 사항
      │ labels: ["qc", severity]
      ▼
      [10. Discord: 알림]
      │ "⚠️ QC 이슈 수동 조치 필요 — {{ summary }}"
```

### Workflow B: 일일 품질 모니터링

```
트리거: Schedule — 매일 09:00 KST (00:00 UTC)

[1. Schedule Trigger]
  │ Cron: 0 0 * * *
  ▼
[2. HTTP Request: QC 통계 (7일)]
  │ GET https://xzawed.xyz/api/v1/admin/qc-stats?days=7
  │ Auth: App Admin API
  ▼
[3. HTTP Request: QC 통계 (이전 7일)]
  │ GET https://xzawed.xyz/api/v1/admin/qc-stats?days=14
  │ → 14일 데이터에서 최근 7일을 빼면 이전 7일 트렌드
  ▼
[4. IF Node: 이슈 있는가?]
  │ 조건: qcPassRate < 0.7 OR commonFailures에 rate > 0.3인 항목 존재
  │
  ├── 이슈 있음:
  │   ▼
  │   [5. HTTP Request: SuperGrok 트렌드 분석]
  │   │ POST https://api.x.ai/v1/chat/completions
  │   │ model: "grok-3"
  │   │ 프롬프트: TREND_ANALYSIS_PROMPT + 현재 vs 이전 통계
  │   ▼
  │   [6. Code Node: 분석 결과 파싱]
  │   ▼
  │   [7. Discord: 일일 리포트]
  │   │ "📊 일일 QC 리포트\n통과율: {{ qcPassRate }}%\n주요 이슈: {{ topIssues }}\n추세: {{ trend }}"
  │
  └── 이슈 없음:
      ▼
      [8. Discord: 정상 리포트]
      │ "✅ QC 일일 리포트 — 정상 (통과율 {{ qcPassRate }}%, 생성 {{ total }}건)"
```

### Workflow C: 코드 리뷰 (Claude CLI PR)

```
트리거: 웹훅 (GitHub PR webhook)

[1. Webhook Trigger: GitHub]
  │ URL: /webhook/github-pr
  │ Events: pull_request (opened, synchronize)
  │ 조건: head.ref가 'claude/'로 시작
  ▼
[2. HTTP Request: PR diff 가져오기]
  │ GET https://api.github.com/repos/xzawed/CustomWebService/pulls/{{ number }}/files
  │ Auth: GitHub PAT
  ▼
[3. Code Node: diff 조합]
  │ 변경된 파일 목록 + patch 내용을 하나의 텍스트로 조합
  │ 10000줄 초과 시 스킵
  ▼
[4. HTTP Request: SuperGrok 코드 리뷰]
  │ POST https://api.x.ai/v1/chat/completions
  │ model: "grok-3"
  │ 프롬프트: CODE_REVIEW_PROMPT + diff 내용
  │ → JSON 반환: { verdict, summary, comments[], securityConcerns[] }
  ▼
[5. Switch Node: verdict]
  │
  ├── 'approve':
  │   ▼
  │   [6. HTTP Request: PR 승인]
  │   │ POST /repos/.../pulls/{{ number }}/reviews
  │   │ Body: { event: "APPROVE", body: "{{ summary }}" }
  │   ▼
  │   [7. Discord: 알림]
  │   │ "✅ PR #{{ number }} 자동 승인 — {{ summary }}"
  │
  └── 'request_changes':
      ▼
      [8. HTTP Request: 변경 요청]
      │ POST /repos/.../pulls/{{ number }}/reviews
      │ Body: { event: "REQUEST_CHANGES", body: comments }
      ▼
      [9. HTTP Request: Claude CLI 재수정 트리거]
      │ repository_dispatch: "claude-fix-review"
      │ client_payload: { prNumber, reviewComments }
      ▼
      [10. Discord: 알림]
      │ "🔄 PR #{{ number }} 수정 요청 → Claude CLI 재작업 중"
```

### Workflow D: 에러 알림 + 분류

```
트리거: 웹훅 (app-events)
조건: type이 '*_FAILED' 패턴

[1. Webhook Trigger]
  │ 조건: type에 'FAILED' 포함
  ▼
[2. HTTP Request: 최근 에러 조회]
  │ (선택) Supabase 직접 쿼리 또는 앱 API
  │ SELECT COUNT(*) FROM platform_events
  │ WHERE type = '{{ type }}' AND created_at > NOW() - INTERVAL '1 hour'
  ▼
[3. IF Node: 에러 임계값 초과?]
  │ 조건: 1시간 내 동일 에러 3건 이상
  │
  ├── 임계값 초과:
  │   ▼
  │   [4. HTTP Request: SuperGrok 에러 분류]
  │   │ model: "grok-3"
  │   │ 프롬프트: ERROR_TRIAGE_PROMPT
  │   │ → { severity, category, isAutoFixable, summary }
  │   ▼
  │   [5. Switch Node: severity]
  │   │
  │   ├── 'critical':
  │   │   [Discord: @here 긴급 알림]
  │   │   [GitHub Issue: critical 라벨]
  │   │
  │   ├── 'high' + autoFixable:
  │   │   [GitHub dispatch: claude-fix]
  │   │   [Discord: 자동 수정 시작 알림]
  │   │
  │   └── 'medium'/'low':
  │       [GitHub Issue 생성]
  │       [Discord: 정보 알림]
  │
  └── 임계값 미달:
      [로그만 기록, 무시]
```

## 8. n8n Cloud 설정 순서

### 1단계: 크리덴셜 등록

n8n Cloud → Settings → Credentials에서:

1. **Header Auth** — `App Admin API`
   - Name: `Authorization`
   - Value: `Bearer <ADMIN_API_KEY와 동일한 값>`

2. **Header Auth** — `xAI SuperGrok`
   - Name: `Authorization`
   - Value: `Bearer <XAI_API_KEY>`

3. **Header Auth** — `GitHub PAT`
   - Name: `Authorization`
   - Value: `Bearer <GITHUB_TOKEN>` (repo + workflow 스코프)

### 2단계: 워크플로우 생성 순서

| 순서 | 워크플로우 | 소요 |
|------|-----------|------|
| 1 | **E. 배포 검증** | 가장 단순, 연결 확인용 |
| 2 | **D. 에러 알림** | 웹훅 수신 + Discord 알림 테스트 |
| 3 | **B. 일일 모니터링** | 스케줄 + Admin API 연동 검증 |
| 4 | **A. QC 자동 수정** | SuperGrok + Claude CLI 전체 루프 |
| 5 | **C. 코드 리뷰** | GitHub 웹훅 + SuperGrok + Claude CLI |

### 3단계: Railway 환경변수 설정
