# 마무리 작업 가이드 (Setup & Launch Checklist)

> 코드는 모두 구현 완료되었습니다.
> 아래 작업은 외부 서비스 설정 및 연동 테스트로, 순서대로 진행하시면 됩니다.

---

## Step 1. Supabase 프로젝트 생성 (10분)

1. https://supabase.com 접속 → GitHub 계정으로 가입
2. **New Project** 클릭
   - Name: `custom-web-service`
   - Region: `Northeast Asia (ap-northeast-1)`
   - Database Password: 강력한 비밀번호 설정 → **메모장에 보관**
3. 프로젝트 생성 완료 대기 (약 2분)
4. **Settings → API** 에서 3개 키 복사:
   - `Project URL` → `.env.local`의 `NEXT_PUBLIC_SUPABASE_URL`
   - `anon public` → `.env.local`의 `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - `service_role` → `.env.local`의 `SUPABASE_SERVICE_ROLE_KEY`

### .env.local 파일 생성
```bash
cp .env.example .env.local
```
복사한 키를 `.env.local`에 붙여넣기

---

## Step 2. DB 테이블 생성 + 시드 데이터 (5분)

1. Supabase Dashboard → **SQL Editor** 열기
2. **첫 번째 실행**: `supabase/migrations/001_initial_schema.sql` 내용 전체 복사 → 실행
   - 10개 테이블 + 인덱스 + RLS 정책이 생성됩니다
3. **두 번째 실행**: `supabase/seed.sql` 내용 전체 복사 → 실행
   - 15개 무료 API + 7개 피처 플래그가 입력됩니다
4. **Table Editor**에서 `api_catalog` 테이블 열어 데이터 확인

---

## Step 3. Google OAuth 설정 (10분)

1. https://console.cloud.google.com 접속
2. **새 프로젝트** 생성: `custom-web-service`
3. 왼쪽 메뉴 → **API 및 서비스** → **OAuth 동의 화면**
   - 유형: `외부`
   - 앱 이름: `CustomWebService`
   - 사용자 지원 이메일: 본인 이메일 입력
   - 스코프 추가: `email`, `profile`, `openid`
   - 저장
4. **사용자 인증 정보** → **+ 사용자 인증 정보 만들기** → **OAuth 클라이언트 ID**
   - 유형: `웹 애플리케이션`
   - 이름: `CustomWebService`
   - 승인된 리디렉션 URI 추가:
     ```
     https://<YOUR_SUPABASE_PROJECT_ID>.supabase.co/auth/v1/callback
     ```
     (Supabase Project URL에서 `https://` 뒤의 `xxxxxxxx.supabase.co` 부분 사용)
   - 만들기 → **Client ID**, **Client Secret** 복사
5. **Supabase Dashboard** → **Authentication** → **Providers** → **Google**
   - Enabled: ON
   - Client ID: 위에서 복사한 값
   - Client Secret: 위에서 복사한 값
   - 저장

---

## Step 4. GitHub OAuth 설정 (5분)

1. https://github.com/settings/developers → **OAuth Apps** → **New OAuth App**
   - Application name: `CustomWebService`
   - Homepage URL: `http://localhost:3000`
   - Authorization callback URL:
     ```
     https://<YOUR_SUPABASE_PROJECT_ID>.supabase.co/auth/v1/callback
     ```
   - Register → **Client ID**, **Client Secret** 복사
2. **Supabase Dashboard** → **Authentication** → **Providers** → **GitHub**
   - Enabled: ON
   - Client ID / Client Secret 입력 → 저장

---

## Step 5. Supabase Auth 추가 설정 (2분)

1. **Supabase Dashboard** → **Authentication** → **URL Configuration**
   - Site URL: `http://localhost:3000`
   - Redirect URLs에 추가:
     ```
     http://localhost:3000/callback
     ```

---

## Step 6. Gemini API 키 발급 (3분)

1. https://aistudio.google.com 접속 (Google 계정 로그인)
2. **Get API key** → **Create API key**
3. 발급된 키 복사 → `.env.local`의 `GEMINI_API_KEY`에 붙여넣기
4. 터미널에서 테스트:
   ```bash
   curl "https://generativelanguage.googleapis.com/v1beta/models/gemini-1.5-flash:generateContent?key=YOUR_KEY" \
     -H 'Content-Type: application/json' \
     -d '{"contents":[{"parts":[{"text":"Hello"}]}]}'
   ```
   → 정상 응답이 오면 성공

---


## Step 8. Vercel 배포 (10분)

1. https://vercel.com 접속 → GitHub 계정으로 로그인
2. **Add New...** → **Project** → GitHub 저장소 `CustomWebService` 선택
3. **Environment Variables** 추가 (`.env.local`의 내용 전부):
   ```
   NEXT_PUBLIC_SUPABASE_URL = (값)
   NEXT_PUBLIC_SUPABASE_ANON_KEY = (값)
   SUPABASE_SERVICE_ROLE_KEY = (값)
   GEMINI_API_KEY = (값)
   NEXT_PUBLIC_APP_URL = https://your-project.vercel.app
   ```
4. **Deploy** 클릭 → 빌드 완료 대기 (약 2분)
5. 배포 URL 접속 → 랜딩 페이지 정상 표시 확인

### Vercel 배포 후 추가 설정

6. 배포 URL 복사 (예: `https://custom-web-service-xxx.vercel.app`)
7. **Supabase Dashboard** → **Authentication** → **URL Configuration**
   - Site URL을 Vercel URL로 변경
   - Redirect URLs에 추가: `https://your-vercel-url.vercel.app/callback`
8. **Google Cloud Console** → OAuth 클라이언트 → 승인된 리디렉션 URI에 추가 (변경 불필요 - Supabase 콜백이므로)
9. **GitHub OAuth App** → Homepage URL을 Vercel URL로 변경

---

## Step 9. 전체 흐름 E2E 확인 (15분)

배포된 사이트에서 전체 흐름을 한 번 돌려봅니다:

```
1. 랜딩 페이지 → "무료로 시작하기" 클릭
2. 로그인 페이지 → Google 또는 GitHub으로 로그인
3. 대시보드 → "서비스 만들러 가기" 클릭
4. 빌더 Step 1 → API 2~3개 선택 (예: Open-Meteo, REST Countries)
5. 빌더 Step 2 → 서비스 설명 입력 (50자 이상)
   예: "서울의 현재 날씨와 한국 국가 정보를 한 화면에 보여주는 깔끔한 대시보드를 만들어주세요.
        온도, 습도, 국기, 수도, 인구 정보를 카드로 보여주세요."
6. 빌더 Step 3 → "생성하기" 클릭 → 프로그레스 바 진행
7. 생성 완료 → "대시보드에서 확인하기"
8. 대시보드 → 생성된 프로젝트 카드 확인
9. "상세 보기" → 프로젝트 상세 페이지
10. "미리보기" → 생성된 웹서비스가 iframe에 렌더링
```

## Step 10. (선택) 추가 개선 작업

모든 핵심 기능이 동작하면 아래를 선택적으로 진행하세요:

| 우선순위 | 작업 | 설명 |
|---------|------|------|
| 높음 | Vercel 토큰 설정 | `.env.local`에 `VERCEL_TOKEN`, `VERCEL_ORG_ID` 추가 → 자동 배포 기능 활성화 |
| 높음 | GitHub Org 생성 | `customwebservice-apps` Organization 생성 → 사용자 서비스 저장소 자동 생성용 |
| 보통 | Discord 알림 | Discord 서버 → 웹훅 생성 → `DISCORD_WEBHOOK_URL` 설정 |
| 보통 | Sentry 연동 | sentry.io 가입 → 프로젝트 생성 → `SENTRY_DSN` 설정 |
| 보통 | 파비콘/로고 | Figma 또는 Canva에서 제작 → `public/` 디렉토리에 배치 |
| 낮음 | 커스텀 도메인 | Vercel → Settings → Domains → 도메인 연결 |
