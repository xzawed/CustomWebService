# 확장성 검토 보고서 (Extensibility Review Report)

## 검토 요약

전체 20개 문서를 확장성/확장 용이성 관점에서 검토한 결과,
**심각 7건, 보통 12건, 낮음 3건** 총 22건의 개선 필요 사항이 발견되었습니다.

현재 설계는 MVP 기능 구현에는 충분하나, 이후 기능 확장 시 대규모 리팩토링이 필요한 구조입니다.
Sprint 1~2에서 기반 구조를 개선하면 이후 스프린트의 확장이 매우 용이해집니다.

## 2. 현재 설계 vs 개선 설계 비교

### 2.1 아키텍처 레이어 (H1)

**현재 (문제)**
```
API Route → Supabase 직접 호출
(검증 + 비즈니스 로직 + DB 접근이 모두 Route 안에 혼재)
```

**개선**
```
API Route (요청/응답 처리)
    ↓
Service Layer (비즈니스 로직)
    ↓
Repository Layer (데이터 접근 추상화)
    ↓
Supabase Client (실제 DB 호출)
```

## 3. DB 스키마 확장성 이슈 상세

### 3.1 멀티 테넌시 (H5)

**현재 스키마**: users → projects (1:N 단순 관계)

**개선 스키마**: organizations → memberships → users, projects

```sql
-- 추가 테이블
CREATE TABLE organizations (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(200) NOT NULL,
    slug VARCHAR(100) UNIQUE NOT NULL,
    plan VARCHAR(20) DEFAULT 'free',        -- free, pro, enterprise
    settings JSONB DEFAULT '{}',
    created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE memberships (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    organization_id UUID NOT NULL REFERENCES organizations(id) ON DELETE CASCADE,
    role VARCHAR(20) DEFAULT 'member',      -- owner, admin, member, viewer
    created_at TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(user_id, organization_id)
);

-- projects 테이블 수정
ALTER TABLE projects ADD COLUMN organization_id UUID REFERENCES organizations(id);
```

**효과**: 팀 협업, 조직별 프로젝트 관리, 향후 유료 플랜 도입 기반

## 5. 누락된 확장성 인프라

### 5.1 이벤트 시스템 (M8)

```typescript
// src/lib/events/eventBus.ts
type DomainEvent =
  | { type: 'PROJECT_CREATED'; payload: { projectId: string; userId: string } }
  | { type: 'CODE_GENERATED'; payload: { projectId: string; version: number; provider: string } }
  | { type: 'DEPLOYMENT_STARTED'; payload: { projectId: string; platform: string } }
  | { type: 'DEPLOYMENT_COMPLETED'; payload: { projectId: string; url: string } }
  | { type: 'DEPLOYMENT_FAILED'; payload: { projectId: string; error: string } }
  | { type: 'API_LIMIT_WARNING'; payload: { service: string; usage: number; limit: number } };

interface EventBus {
  emit(event: DomainEvent): void;
  on(type: DomainEvent['type'], handler: (event: DomainEvent) => void): void;
  off(type: DomainEvent['type'], handler: (event: DomainEvent) => void): void;
}
```

**효과**: 분석, 알림, 로깅, 웹훅 등을 핵심 로직 수정 없이 추가 가능
