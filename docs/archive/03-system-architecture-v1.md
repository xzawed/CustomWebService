## 2. 기술 스택

### Frontend
| 구분 | 기술 | 선정 이유 |
|------|------|-----------|
| 프레임워크 | **Next.js 16 (App Router)** | SSR/SSG 지원, Railway 배포 |
| 언어 | **TypeScript** | 타입 안정성, 개발 생산성 |
| UI 라이브러리 | **Tailwind CSS** | 빠른 스타일링, 번들 최적화 |
| 컴포넌트 | **shadcn/ui** | 무료, 커스터마이징 자유도 높음 |
| 드래그 앤 드롭 | **@dnd-kit/core** | 경량, 접근성 우수, 무료 |
| 상태 관리 | **Zustand** | 경량, 간단한 API |
| 폼 관리 | **React Hook Form + Zod** | 검증 내장, 성능 우수 |

### Backend
| 구분 | 기술 | 선정 이유 |
|------|------|-----------|
| API 서버 | **Next.js API Routes** | 프론트와 통합, Railway 배포 |
| 런타임 | **Node.js** | Railway 무료 Trial |
| 인증 | **Supabase Auth** | 무료 티어 충분, 소셜 로그인 |

### 데이터베이스
| 구분 | 기술 | 선정 이유 |
|------|------|-----------|
| 메인 DB | **Supabase (PostgreSQL)** | 무료: 500MB, 50K MAU |
| 파일 저장 | **Supabase Storage** | 무료: 1GB |

### AI / 코드 생성
| 구분 | 기술 | 선정 이유 |
|------|------|-----------|
| 코드 생성 | **Claude API** | 고품질 코드 생성 능력 |
| 대안 | **OpenAI API (GPT)** | 무료 크레딧 활용 가능 시 |
| 오프라인 대안 | **Ollama (로컬 LLM)** | 완전 무료, 서버 필요 |

### 배포 / 호스팅
| 구분 | 기술 | 무료 한도 |
|------|------|-----------|
| 메인 플랫폼 | **Vercel** | 100GB 대역폭/월, Serverless 함수 |
| 생성 서비스 배포 | **Vercel** | 프로젝트당 무료 |
| 대안 1 | **Netlify** | 100GB 대역폭/월 |
| 대안 2 | **GitHub Pages** | 정적 사이트 무한 |
| 소스 관리 | **GitHub** | 무료 Private 저장소 |

### 모니터링
| 구분 | 기술 | 선정 이유 |
|------|------|-----------|
| 에러 추적 | **Sentry (무료 티어)** | 5K 이벤트/월 무료 |
| 분석 | **Vercel Analytics** | 기본 무료 |
| 업타임 | **UptimeRobot** | 50개 모니터 무료 |

## 3. 데이터 흐름

### 웹서비스 생성 프로세스

```
1. [사용자] API 선택 + 컨텍스트 입력
       │
       ▼
2. [Backend] 입력 검증 & 프롬프트 구성
       │
       ▼
3. [AI Service] 프롬프트 → 코드 생성
       │  - HTML/CSS/JS 또는 React 코드
       │  - API 연동 코드 포함
       │  - 반응형 디자인 적용
       ▼
4. [Backend] 생성 코드 검증 & 후처리
       │  - 보안 검사
       │  - API 키 환경변수 처리
       │  - 코드 포맷팅
       ▼
5. [Deploy Manager] GitHub 저장소 생성 → 배포 플랫폼 연동
       │
       ▼
6. [사용자] 배포 URL 수신 & 서비스 사용
```

## 5. 보안 아키텍처

### API 키 관리
- 사용자 API 키: Supabase에 암호화 저장
- 플랫폼 API 키: 환경변수로 서버사이드에서만 접근
- 생성된 서비스의 API 키: 배포 플랫폼 환경변수로 주입

### 생성 코드 보안
- XSS 방지: 사용자 입력 이스케이프 처리
- CSP 헤더 자동 적용
- 외부 스크립트 로드 제한
- API 키 노출 방지 (서버사이드 프록시)
