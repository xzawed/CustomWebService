# 서비스 기획서 (Product Planning Document)

## 1. 서비스 정의

### 1.1 서비스명
**CustomWebService** - 무료 API 기반 맞춤형 웹서비스 자동 생성 플랫폼

### 1.2 서비스 한 줄 정의
> "무료 API를 골라 담고, 원하는 서비스를 설명하면, AI가 웹서비스를 만들어 바로 배포해주는 플랫폼"

### 1.3 해결하려는 문제
| 문제 | 현재 상황 | 우리의 해결 |
|------|-----------|------------|
| 개발 능력 부재 | 아이디어가 있어도 개발을 못 함 | AI가 코드를 대신 생성 |
| API 접근 장벽 | API 문서 읽기, 연동 개발이 어려움 | 클릭만으로 API 선택 및 자동 연동 |
| 비용 부담 | 서버, 도메인, API 비용 발생 | 100% 무료 운영 |
| 배포 복잡성 | 서버 설정, 배포 과정이 복잡 | 원클릭 자동 배포 |

### 1.4 핵심 성공 지표 (KPI)
| 지표 | 목표 (출시 후 3개월) |
|------|---------------------|
| 가입 사용자 수 | 500명 |
| 생성된 서비스 수 | 1,000개 |
| 서비스 생성 완료율 | 70% (시작 대비 완료) |
| 평균 생성 소요 시간 | 2분 이내 |
| 사용자 재방문율 | 30% |

## 2. 사용자 정의

### 2.1 페르소나

#### 페르소나 A: 김기획 (비개발자 기획자)
- **나이/직업**: 32세, 스타트업 기획자
- **기술 수준**: HTML 기초만 아는 비개발자
- **니즈**: 빠르게 프로토타입을 만들어서 팀에 보여주고 싶음
- **고충**: 개발자에게 매번 요청하면 시간이 너무 오래 걸림
- **시나리오**: "환율 API랑 국가 정보 API를 조합해서 여행 환율 계산기를 만들고 싶어"

#### 페르소나 B: 이소상 (소규모 사업자)
- **나이/직업**: 45세, 동네 카페 사장
- **기술 수준**: 컴퓨터 기본 사용만 가능
- **니즈**: 가게 주변 날씨와 공지를 보여주는 간단한 웹페이지
- **고충**: 웹사이트 제작 업체 비용이 부담됨
- **시나리오**: "날씨 API로 가게 앞에 놓을 태블릿용 날씨 화면을 만들고 싶어"

#### 페르소나 C: 박학생 (대학생 사이드 프로젝트)
- **나이/직업**: 22세, 컴공과 2학년
- **기술 수준**: 기초 프로그래밍 가능, 웹 개발은 미숙
- **니즈**: 포트폴리오용 프로젝트를 빠르게 만들고 싶음
- **고충**: API 연동과 배포를 혼자 하기 어려움
- **시나리오**: "영화 API랑 뉴스 API를 조합해서 엔터테인먼트 대시보드를 만들고 싶어"

### 3.2 서비스 빌더 (핵심 기능)

#### 3.2.1 Step 1: API 선택

**화면 레이아웃**
- 좌측(60%): API 카탈로그 영역
- 우측(40%): 선택된 API 영역
- 상단: 스텝 인디케이터 (1/3)

**카탈로그 영역 상세**

| 요소 | 동작 |
|------|------|
| 검색바 | 실시간 필터링 (debounce 300ms) |
| 카테고리 탭 | 전체/날씨/뉴스/금융/지도/번역/이미지/데이터/유틸/엔터 |
| API 카드 | 이름, 설명, 카테고리 뱃지, 호출 제한, 인증 방식 표시 |
| 카드 체크박스 | 클릭 시 우측 선택 영역에 추가 |
| 카드 드래그 | 우측 영역으로 드래그 시 추가 |
| 카드 클릭 | 상세 모달 열기 (엔드포인트, 응답 예시, 문서 링크) |

**선택 영역 상세**

| 요소 | 동작 |
|------|------|
| 선택된 API 카드 | 미니 카드 형태, X 버튼으로 제거 |
| 드롭 존 | 드래그 시 하이라이트 "여기에 놓으세요" |
| 카운터 | "선택된 API: N/5개" |
| 추천 조합 | 선택된 API 기반 궁합 좋은 API 추천 (선택사항) |
| 다음 버튼 | 최소 1개 선택 시 활성화 |

**API 상세 모달**

```
┌─ OpenWeatherMap ──────────────────────────────┐
│                                                │
│  🌤 OpenWeatherMap                             │
│  현재 날씨, 5일 예보, 대기질 정보를 제공합니다      │
│                                                │
│  ─── 기본 정보 ───                              │
│  카테고리: 날씨                                  │
│  인증: API Key (무료 가입)                       │
│  호출 제한: 1,000회/일                           │
│  응답 형식: JSON                                │
│                                                │
│  ─── 사용 가능 엔드포인트 ───                     │
│  GET /weather     현재 날씨 조회                 │
│  GET /forecast    5일 예보 조회                  │
│  GET /air_pollution 대기질 조회                  │
│                                                │
│  ─── 응답 예시 ───                              │
│  { "temp": 20, "description": "맑음", ... }    │
│                                                │
│  [📄 공식 문서]          [✅ 선택하기]            │
└────────────────────────────────────────────────┘
```

**비즈니스 규칙**
- 최소 1개, 최대 5개 API 선택 가능
- 비활성(장애) API는 선택 불가 (회색 처리)
- 이미 선택된 API는 카탈로그에서 체크 표시

#### 3.2.2 Step 2: 서비스 설명 (컨텍스트 입력)

**화면 레이아웃**
- 상단: 선택된 API 요약 바 (수정 가능)
- 중앙: 컨텍스트 입력 영역
- 하단: 가이드/템플릿 영역

**컨텍스트 입력 영역**

| 요소 | 상세 |
|------|------|
| 레이블 | "어떤 서비스를 만들고 싶나요?" |
| 플레이스홀더 | "예: 여행자를 위한 환율 계산기를 만들고 싶어요. 국가를 선택하면..." |
| 글자 수 | 실시간 카운터 "123 / 2,000자" |
| 최소 검증 | 50자 미만 시 "좀 더 자세히 설명해주세요" 안내 |
| 자동 저장 | 입력 중 로컬 스토리지에 자동 저장 (새로고침 대비) |

**가이드 질문 (토글 펼침)**
```
💡 이런 내용을 포함하면 더 좋은 서비스가 만들어져요:

1. 누가 사용하나요?
   예) "여행을 준비하는 사람들이 사용해요"

2. 어떤 정보를 보여주고 싶나요?
   예) "환율 정보와 국가별 통화를 보여주고 싶어요"

3. 사용자가 뭔가 입력하거나 선택하나요?
   예) "국가를 드롭다운으로 선택하고, 금액을 입력하면 변환해줘요"

4. 화면 분위기는 어떤 느낌이면 좋겠나요?
   예) "깔끔하고 모던한 느낌", "다크 테마", "컬러풀하고 재미있는 느낌"

5. 특별히 원하는 기능이 있나요?
   예) "실시간 업데이트", "차트로 시각화", "비교 기능"
```

**템플릿 (퀵 선택)**

| 템플릿 | 자동 채워지는 컨텍스트 |
|--------|----------------------|
| 📊 대시보드 | "선택한 API의 데이터를 한 화면에 카드와 차트로 보여주는 대시보드를 만들어주세요. 실시간으로 데이터가 업데이트되면 좋겠어요." |
| 🔢 계산기/변환기 | "사용자가 값을 입력하면 API 데이터를 기반으로 계산/변환해주는 도구를 만들어주세요." |
| 🔍 정보 조회 | "검색어를 입력하거나 카테고리를 선택하면 API에서 정보를 가져와 카드 형태로 보여주는 서비스를 만들어주세요." |
| 🖼 갤러리 | "API에서 가져온 이미지나 콘텐츠를 그리드 형태로 보여주는 갤러리를 만들어주세요." |
| 📰 피드 | "API에서 가져온 콘텐츠를 뉴스피드처럼 스크롤로 볼 수 있게 만들어주세요." |
| 🗺 지도 서비스 | "지도 위에 API 데이터를 마커나 오버레이로 표시하는 서비스를 만들어주세요." |

**비즈니스 규칙**
- 템플릿 선택 시 텍스트 자동 채움 (사용자가 수정 가능)
- 50자 미만이면 "생성하기" 버튼 비활성화
- 2,000자 초과 시 입력 제한
- 선택된 API와 무관한 내용도 허용 (AI가 판단)

```
      │ 대시보드      │
      │ /dashboard   │
      └──────────────┘
```

## 5. 예외 처리 시나리오

| 시나리오 | 대응 |
|---------|------|
| AI 코드 생성 실패 | 에러 메시지 + "다시 시도" 버튼 + 다른 AI로 폴백 |
| 선택한 API 장애 | API 카드에 장애 표시 + 대체 API 추천 |
| 배포 실패 | 에러 원인 표시 + 다른 플랫폼으로 재시도 |
| 무료 한도 초과 (AI) | "잠시 후 다시 시도" + 예상 복구 시간 안내 |
| 무료 한도 초과 (호스팅) | 대체 플랫폼 자동 선택 |
| 생성 코드 보안 이슈 | 자동 수정 후 재검증 |
| 네트워크 오류 | 자동 재시도 (3회) + 실패 시 안내 |
| 브라우저 새로고침 | 로컬 스토리지에서 진행 상태 복원 |
| 동시 대량 요청 | 큐잉 + "앞에 N명 대기 중" 안내 |

## 6. 추가 구현 기능

초기 기획 이후 다음 기능들이 추가 구현되었습니다.

### 6.1 롤백 기능
- `POST /api/v1/projects/:id/rollback` 엔드포인트
- 생성된 코드를 이전 버전으로 되돌리기 가능
- `generated_codes` 테이블의 version 컬럼 + `projects.current_version`으로 관리

### 6.2 코드 생성 템플릿 시스템
- `ICodeTemplate` 인터페이스 기반 확장 가능한 템플릿 구조
- `TemplateRegistry`에 등록/조회/매칭
- 현재 3종: DashboardTemplate, CalculatorTemplate, GalleryTemplate
- API 적합도 판단(`matchScore`) 후 최적 템플릿 자동 선택

### 6.3 피처 플래그 시스템
- `feature_flags` 테이블 기반 (DB 캐싱)
- 7개 기본 플래그:
  - `enable_dark_mode`: 다크 테마
  - `enable_code_viewer`: 코드 뷰어
  - `enable_ollama_fallback`: Ollama 폴백
  - `enable_template_system`: 템플릿 시스템
  - `enable_multi_language`: 다국어 지원
  - `enable_team_features`: 팀 기능
  - `enable_advanced_prompt`: 고급 프롬프트
- `rules` JSONB로 조건부 활성화 지원

### 6.4 조직/팀 기능 (멀티 테넌시)
- `organizations` 테이블: 조직 단위 관리 (name, slug, plan, settings)
- `memberships` 테이블: 조직-사용자 매핑 (역할: owner/admin/member/viewer)
- `projects.organization_id`: 조직 프로젝트 지원
- RLS 정책으로 조직 멤버만 프로젝트 조회 가능

### 6.5 도메인 이벤트 시스템
- `EventBus`: 이벤트 발행/구독 (observer 패턴)
- `event_log` 테이블: 감사 로그 저장
- 9종 도메인 이벤트 타입 (CODE_GENERATED, DEPLOYMENT_COMPLETED 등)
- 핵심 코드 수정 없이 부가 기능 추가 가능

### 6.6 법적 페이지
- `/terms`: 이용약관
- `/privacy`: 개인정보처리방침
- `/disclaimer`: 면책 조항
- 푸터에 법적 문서 링크 표시

## 7. 용어 정의

| 용어 | 정의 |
|------|------|
| API 카탈로그 | 플랫폼에서 제공하는 무료 API 목록 |
| 빌더 | 서비스 생성을 위한 3단계 위자드 인터페이스 |
| 컨텍스트 | 사용자가 입력하는 서비스 설명 텍스트 |
| 생성 | AI가 코드를 만드는 과정 |
| 배포 | 생성된 코드를 호스팅 플랫폼에 올리는 과정 |
| 서비스 | 생성/배포 완료되어 URL로 접근 가능한 웹사이트 |
| 프로젝트 | 하나의 서비스 생성 건 (선택 API + 컨텍스트 + 코드 + 배포 정보) |
