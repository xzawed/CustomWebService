# 요구사항 정의서 (Requirements Specification)

## 1. 기능 요구사항 (Functional Requirements)

### FR-01. 무료 API 카탈로그
| 항목 | 설명 |
|------|------|
| ID | FR-01 |
| 우선순위 | 필수 |
| 설명 | 영구 무료 API 목록을 카테고리별로 관리하고 표시 |

- FR-01-1: API를 카테고리별로 분류하여 표시 (날씨, 뉴스, 금융, 지도, 번역 등)
- FR-01-2: 각 API에 대한 설명, 제공 데이터, 호출 제한, 인증 방식 정보 표시
- FR-01-3: API 검색 기능 (이름, 카테고리, 키워드)
- FR-01-4: API 상태(정상/장애) 실시간 모니터링 표시
- FR-01-5: 영구 무료 여부 검증 기준 명시 및 관리

### FR-02. API 선택 인터페이스
| 항목 | 설명 |
|------|------|
| ID | FR-02 |
| 우선순위 | 필수 |

- FR-02-1: 드래그 앤 드롭으로 API를 "내 선택" 영역으로 이동

- FR-02-6: 최대 선택 가능 API 수 제한 설정

### FR-03. 서비스 컨텍스트 입력
| 항목 | 설명 |
|------|------|
| ID | FR-03 |
| 우선순위 | 필수 |
| 설명 | 사용자가 원하는 서비스를 자연어로 설명 |

- FR-03-1: 텍스트 입력 필드 제공 (최소 50자, 최대 2000자)

- FR-03-3: 예시 컨텍스트 템플릿 제공
- FR-03-4: 선택된 API 기반 자동 추천 컨텍스트
- FR-03-5: 입력 내용 유효성 검증

## 2. 비기능 요구사항 (Non-Functional Requirements)

### NFR-01. 성능
- 페이지 로드 시간: 3초 이내
- 웹서비스 생성 시간: 3분 이내
- 배포 완료 시간: 생성 후 2분 이내
- 동시 생성 요청 처리: 최소 10건

### NFR-02. 사용성
- 비개발자가 5분 이내에 첫 서비스 생성 가능
- 모바일/태블릿 반응형 지원
- 한국어 UI 기본 제공
- 온보딩 가이드/튜토리얼 제공

### NFR-03. 보안
- API 키 안전한 저장 및 관리
- 생성된 코드 XSS/인젝션 방지
- HTTPS 필수 적용
- 사용자 데이터 최소 수집

### NFR-04. 가용성
- 서비스 가동률 99% 이상 목표
- 무료 호스팅 제한 내 운영
- 장애 시 사용자 알림

### NFR-05. 비용
- 모든 인프라 무료 티어 내 운영
- API 호출 비용 제로
- 호스팅 비용 제로
- 도메인 비용 제로 (서브도메인 활용)
