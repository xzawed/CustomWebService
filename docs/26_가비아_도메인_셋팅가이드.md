# 가비아 도메인 셋팅 가이드 (xzawed.xyz)

> 도메인: **xzawed.xyz** (가비아에서 구매)
> 목표: `xzawed.xyz`로 서비스 접속 + `xxx.xzawed.xyz`로 사용자 서비스 게시
>
> 이 가이드는 웹 브라우저에서 클릭만으로 따라할 수 있습니다.
> 예상 소요 시간: 약 30분

---

## 전체 흐름 요약

```
가비아 DNS 설정 (작업 1)
    ↓
Railway 도메인 연결 (작업 2)
    ↓
Railway 환경변수 추가 (작업 3)
    ↓
Supabase 로그인 URL 수정 (작업 4)
    ↓
DB에 API 데이터 넣기 (작업 5)
    ↓
정상 동작 확인 (작업 6)
```

---

## 작업 1: 가비아 DNS 설정 (10분)

> 가비아에게 "xzawed.xyz 도메인으로 들어오는 사람은 Railway 서버로 보내줘"라고 알려주는 작업입니다.

### 1단계: 가비아 관리 페이지 열기

**1-1.** 웹 브라우저를 열고 주소창에 입력합니다:

```
https://dns.gabia.com
```

또는 아래 경로로 접근할 수도 있습니다:
- 가비아 홈(www.gabia.com) → 로그인
- 상단 메뉴 **"My가비아"** 클릭
- **"도메인 관리"** 클릭
- 도메인 목록에서 **"xzawed.xyz"** 오른쪽의 **"관리"** 버튼 클릭
- **"DNS 관리"** 탭 클릭

### 2단계: DNS 레코드 추가 화면 열기

**1-2.** DNS 설정 페이지가 열리면, 상단에 **"DNS 설정"** 또는 **"DNS 레코드 수정"** 버튼이 있습니다. 이것을 클릭합니다.

**1-3.** 현재 등록된 레코드 목록이 보입니다. 여기에 새 레코드를 추가할 겁니다.

### 3단계: 첫 번째 레코드 추가 (메인 도메인)

> `xzawed.xyz`로 접속했을 때 Railway 서버로 연결합니다.

**1-4.** **"레코드 추가"** 버튼을 클릭합니다.

**1-5.** 아래와 같이 입력합니다:

```
┌──────────────────────────────────────────────────────┐
│  타입:     CNAME  (드롭다운에서 선택)                  │
│                                                      │
│  호스트:   @                                          │
│           (또는 빈칸 — 가비아에 따라 다를 수 있음)       │
│                                                      │
│  값/위치:  customwebservice-production.up.railway.app  │
│           (이 주소를 정확히 복사해서 붙여넣으세요)       │
│                                                      │
│  TTL:     3600  (기본값 그대로 두세요)                 │
└──────────────────────────────────────────────────────┘
```

**주의사항:**
- "값/위치" 끝에 마침표(.)가 자동으로 붙을 수 있습니다. 정상입니다.
- 가비아에서 "호스트"란에 `@`를 입력하면 `xzawed.xyz` 자체를 의미합니다.

**1-6.** **"확인"** 또는 **"저장"** 을 클릭합니다.

---

**만약 가비아에서 루트 도메인(@)에 CNAME을 설정할 수 없다는 안내가 나오면:**

가비아는 루트 도메인에 CNAME 대신 **A 레코드**를 요구할 수 있습니다.
이 경우 아래 대안을 사용하세요:

**대안 A — 가비아 "웹 포워딩" 사용:**
1. DNS 설정 대신 **"도메인 포워딩"** 메뉴를 찾습니다
2. `xzawed.xyz` → `https://www.xzawed.xyz` 로 포워딩 설정
3. 아래 4단계에서 `www` 서브도메인 CNAME을 추가합니다

**대안 B — www 서브도메인만 사용:**
1. 루트(@) CNAME 대신, `www` CNAME만 추가합니다
2. 서비스 URL이 `https://www.xzawed.xyz`가 됩니다

---

### 4단계: 두 번째 레코드 추가 (와일드카드 서브도메인)

> `xxx.xzawed.xyz` 형태로 사용자 서비스를 게시할 수 있게 합니다.

**1-7.** 다시 **"레코드 추가"** 버튼을 클릭합니다.

**1-8.** 아래와 같이 입력합니다:

```
┌──────────────────────────────────────────────────────┐
│  타입:     CNAME                                      │
│                                                      │
│  호스트:   *  (별표 하나)                              │
│                                                      │
│  값/위치:  customwebservice-production.up.railway.app  │
│                                                      │
│  TTL:     3600                                       │
└──────────────────────────────────────────────────────┘
```

**1-9.** **"확인"** 또는 **"저장"**을 클릭합니다.

### 5단계: 설정 확인

**1-10.** DNS 레코드 목록에 다음 두 줄이 보이면 성공입니다:

```
타입     호스트    값/위치                                          TTL
────     ────     ────────────────────────────────────────          ────
CNAME    @        customwebservice-production.up.railway.app       3600
CNAME    *        customwebservice-production.up.railway.app       3600
```

**1-11.** **"저장"** 또는 **"적용"** 버튼이 따로 있다면 반드시 클릭하세요.

> 가비아 DNS 변경은 보통 **10분~1시간** 이내에 반영됩니다.
> 드물게 최대 24시간 걸릴 수 있습니다.

### 제대로 됐는지 확인하는 방법

**1-12.** 10분 정도 기다린 후, 아래 사이트에서 확인할 수 있습니다:

```
https://www.whatsmydns.net/#CNAME/xzawed.xyz
```

이 사이트에 접속하면 전 세계 여러 서버에서 DNS 조회 결과를 보여줍니다.
`customwebservice-production.up.railway.app`이 보이면 DNS 설정이 성공한 것입니다.

## 작업 2: Railway에 도메인 연결 (5분)

> Railway에게 "xzawed.xyz 도메인은 우리 서비스야"라고 알려주는 작업입니다.

**2-1.** 웹 브라우저에서 Railway를 엽니다:
```
https://railway.app/dashboard
```

**2-2.** 프로젝트 목록에서 **CustomWebService** 프로젝트를 클릭합니다.

**2-3.** 화면에 보이는 서비스 블록(보라색/파란색 박스)을 클릭합니다.

**2-4.** 오른쪽 패널에서 **"Settings"** 탭을 클릭합니다.

**2-5.** 아래로 스크롤하여 **"Networking"** 섹션을 찾습니다.

**2-6.** **"Custom Domain"** 영역에서 입력창에 첫 번째 도메인을 입력합니다:
```
xzawed.xyz
```
Enter를 누릅니다.

**2-7.** 다시 입력창에 두 번째 도메인을 입력합니다:
```
*.xzawed.xyz
```
Enter를 누릅니다.

**2-8.** 각 도메인 옆에 상태 표시를 확인합니다:

| 상태 | 의미 |
|------|------|
| Pending | DNS가 아직 전파 중. 10분~1시간 기다리세요 |
| Active (초록색) | 연결 성공! |
| SSL Certificate Issued | HTTPS 인증서 발급 완료 |

**2-9.** 5~10분 후 페이지를 새로고침(F5)하면 상태가 업데이트됩니다.

## 작업 3: Railway 환경변수 추가 (3분)

> 우리 서비스에게 "네 도메인은 xzawed.xyz야"라고 알려주는 작업입니다.

**3-1.** 작업 2에서 열어둔 Railway 서비스 패널에서 **"Variables"** 탭을 클릭합니다.

**3-2.** 기존 환경변수 목록이 보입니다.

**3-3.** `NEXT_PUBLIC_ROOT_DOMAIN` 변수가 이미 있는지 확인합니다:
- **이미 있다면**: 값을 클릭해서 `xzawed.xyz`로 수정합니다
- **없다면**: **"+ New Variable"** 를 클릭하고 아래처럼 입력합니다

```
이름 (왼쪽 칸):   NEXT_PUBLIC_ROOT_DOMAIN
값 (오른쪽 칸):    xzawed.xyz
```

**3-4.** Enter를 누릅니다.

**3-5.** `NEXT_PUBLIC_APP_URL` 변수도 확인합니다:
- **이미 있다면**: 값을 `https://xzawed.xyz`로 수정합니다
- **없다면**: **"+ New Variable"** 를 클릭하고 아래처럼 입력합니다

```
이름:   NEXT_PUBLIC_APP_URL
값:     https://xzawed.xyz
```

**3-6.** 화면 아래쪽에 **"Deploy"** 버튼이 나타나면 클릭합니다.
- 환경변수 변경 후 재배포해야 반영됩니다
- 재배포는 1~3분 정도 걸립니다

**3-7.** **"Deployments"** 탭에서 최신 배포가 초록색 **"Active"** 상태가 되면 완료입니다.

## 작업 4: Supabase 로그인 URL 수정 (5분)

> Google/GitHub 로그인 후 사용자가 돌아올 주소를 xzawed.xyz로 바꿔줍니다.

**4-1.** 웹 브라우저에서 Supabase를 엽니다:
```
https://supabase.com/dashboard
```

**4-2.** 프로젝트(custom-web-service)를 클릭합니다.

**4-3.** 왼쪽 메뉴에서 **Authentication** (자물쇠 아이콘)을 클릭합니다.

**4-4.** 상단 탭에서 **"URL Configuration"**을 클릭합니다.

**4-5.** **Site URL** 칸을 찾아서 다음으로 변경합니다:
```
https://xzawed.xyz
```

**4-6.** **"Save"** 버튼을 클릭합니다.

**4-7.** 아래로 스크롤하여 **"Redirect URLs"** 섹션을 찾습니다.

**4-8.** **"Add URL"** 버튼을 클릭하고 아래 URL을 추가합니다:
```
https://xzawed.xyz/callback
```
"Add" 클릭.

**4-9.** 최종적으로 Redirect URLs 목록에 **최소한 이것들**이 있어야 합니다:

```
http://localhost:3000/callback                                    ← 기존 (삭제하지 마세요)
https://customwebservice-production.up.railway.app/callback       ← 기존 또는 추가
https://xzawed.xyz/callback                                      ← 새로 추가
```

## 작업 5: DB에 54개 API 데이터 넣기 (3분)

> 이 작업은 이전 `25_운영자_수행가이드.md`의 "작업 1"과 동일합니다.

**5-1.** Supabase 대시보드 → **SQL Editor** (왼쪽 메뉴, `</>` 아이콘)

**5-2.** GitHub에서 seed.sql 파일 내용을 복사합니다:
```
https://github.com/xzawed/CustomWebService/blob/master/supabase/seed.sql
```
(페이지에서 "Raw" 버튼 클릭 → 전체 선택(Ctrl+A) → 복사(Ctrl+C))

**5-3.** SQL Editor에 붙여넣기(Ctrl+V) → **"Run"** 버튼 클릭

**5-4.** "Success" 메시지가 나오면 완료.

**5-5.** 확인: Table Editor → api_catalog → 54개 행이면 정상.

## 작업 6: 정상 동작 확인 (5분)

### DNS가 반영된 후 확인 (10분~1시간 후)

**6-1.** 브라우저 주소창에 입력:
```
https://xzawed.xyz
```

확인:
- [ ] 랜딩 페이지가 보인다 (다크 배경, "코딩 없이 만드는 나만의 웹서비스")
- [ ] 주소창에 자물쇠 아이콘이 있다 (HTTPS)

**6-2.** API 카탈로그 확인:
```
https://xzawed.xyz/catalog
```

확인:
- [ ] API 카드들이 보인다 (54개)
- [ ] 한국어 설명이 보인다
- [ ] 아무 API 카드 클릭 시 상세 모달이 에러 없이 열린다

**6-3.** 로그인 확인:
- 오른쪽 위 "로그인" 클릭
- Google 또는 GitHub 로그인 진행

확인:
- [ ] 로그인 화면이 나온다
- [ ] 로그인 후 대시보드(`/dashboard`)로 이동한다

**6-4.** 서브도메인 확인 (서비스 게시 후):
```
https://[slug].xzawed.xyz
```

확인:
- [ ] 게시된 서비스 HTML이 보인다
- [ ] 없는 slug 입력 시 404 페이지가 보인다

## 문제 해결

| 문제 | 원인 | 해결 |
|------|------|------|
| 사이트가 안 열린다 | DNS 아직 미전파 | 10분~1시간 기다리기. whatsmydns.net에서 확인 |
| "이 사이트에 연결할 수 없음" | CNAME 설정 오류 | 가비아 DNS에서 값이 정확한지 다시 확인 |
| "안전하지 않은 사이트" 경고 | SSL 인증서 미발급 | Railway 도메인 상태가 Active인지 확인. 10분 대기 |
| 로그인 후 에러 페이지 | Redirect URL 미설정 | Supabase에서 `https://xzawed.xyz/callback` 추가 확인 |
| 서브도메인 접속 안 됨 | 와일드카드 DNS 미설정 | 가비아에 `*` CNAME이 있는지 확인 |
| API가 15개만 보인다 | seed.sql 미실행 | 작업 5 수행 |
| "502 Bad Gateway" | Railway 재배포 중 | 2~3분 기다리기 |
